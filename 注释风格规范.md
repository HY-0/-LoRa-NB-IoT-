# 项目注释风格规范

## 概述
本文档定义了项目中统一的代码注释风格规范，旨在提高代码的可读性、可维护性和文档生成的一致性。

## 1. 文件头注释

每个源文件（.c/.h）应在文件开头包含文件头注释：

```c
/**
 * @file    文件名
 * @brief   文件功能描述
 * @author  作者
 * @date    创建日期
 * @version 版本号
 */
```

示例：
```c
/**
 * @file    sht30.c
 * @brief   SHT30温湿度传感器驱动
 * @author  Lora项目组
 * @date    2024-01-15
 * @version 1.0.0
 */
```

## 2. 函数注释

所有函数（包括静态函数）都应使用Doxygen风格的注释：

### 2.1 公共函数
```c
/**
 * @brief   函数功能描述
 * @param   参数名 - 参数描述
 * @retval  返回值描述
 */
```

### 2.2 示例
```c
/**
 * @brief   初始化SHT30传感器
 * @param   无
 * @retval  SHT30_EOK   - 初始化成功
 *          SHT30_ERROR - 初始化失败
 */
uint8_t sht30_init(void)
{
    // 函数实现
}
```

### 2.3 多参数函数
```c
/**
 * @brief   读取温湿度数据
 * @param   temp_celsius   - 温度值指针（单位：摄氏度）
 * @param   humi_percent   - 湿度值指针（单位：百分比）
 * @retval  SHT30_EOK      - 读取成功
 *          SHT30_ECRC     - CRC校验失败
 *          SHT30_ETIMEOUT - 读取超时
 */
uint8_t sht30_read_humiture(float *temp_celsius, float *humi_percent);
```

## 3. 宏定义注释

### 3.1 简单宏
```c
#define SHT30_I2C_ADDR    0x44  /* SHT30 I2C地址 */
```

### 3.2 复杂宏或多行宏
```c
/**
 * @brief   SHT30测量命令（高重复性）
 * @note    使用高重复性模式可获得最精确的测量结果
 */
#define SHT30_CMD_MEAS_HIGH_REP  0x2C06
```

### 3.3 枚举注释
```c
/**
 * @brief   测量模式枚举
 */
typedef enum {
    BH1750_MODE_CONT_H_RES = 0,   /* 连续高分辨率模式 */
    BH1750_MODE_CONT_H_RES2,      /* 连续高分辨率模式2 */
    BH1750_MODE_CONT_L_RES,       /* 连续低分辨率模式 */
} BH1750_Mode_t;
```

## 4. 变量注释

### 4.1 全局变量
```c
/**
 * @brief   当前测量命令（默认高重复性）
 * @note    可通过配置函数修改
 */
static uint16_t g_sht30_meas_cmd = SHT30_CMD_MEAS_HIGH_REP;
```

### 4.2 局部变量（重要变量）
```c
uint8_t ret;          /* 函数返回值 */
float temperature;    /* 温度值 */
float humidity;       /* 湿度值 */
```

## 5. 代码块注释

### 5.1 节标题注释
```c
/* ==================== 内部辅助函数 ==================== */
```

### 5.2 代码段注释
```c
/* 初始化I2C硬件 */
sht30_i2c_init(SHT30_I2C_SPEED);

/* 等待传感器上电稳定 */
delay_ms(10);
```

## 6. 单行注释

### 6.1 行尾注释
```c
delay_ms(500);  /* 每500ms刷新一次 */
```

### 6.2 独立行注释
```c
/* 显示标题（仿lora_demo风格） */
oled_clear(&oled);
```

### 6.3 临时注释（调试用）
```c
// TODO: 需要添加错误处理
// FIXME: 这里存在潜在的缓冲区溢出风险
// NOTE: 这个函数在中断中调用，必须保持简短
```

## 7. 特殊注释标签

### 7.1 待办事项
```c
// TODO: [作者] 描述待完成的工作
// TODO: 需要添加参数校验
```

### 7.2 注意事项
```c
// NOTE: 这个函数不能在中断中调用
// WARNING: 修改此参数可能影响系统稳定性
```

### 7.3 修复标记
```c
// FIXME: 临时解决方案，需要重构
// BUG: 在特定条件下会出现内存泄漏
```

## 8. 格式规范

### 8.1 缩进
- 使用4个空格进行缩进
- 注释与代码保持相同缩进级别

### 8.2 对齐
- 多行注释的星号(*)对齐
- 参数描述对齐

### 8.3 语言
- 使用中文进行注释
- 专业术语可使用英文

## 9. 工具支持

### 9.1 Doxygen配置
项目已配置Doxygen支持，可通过以下命令生成文档：
```bash
doxygen Doxyfile
```

### 9.2 代码格式化
使用.clang-format文件保持代码格式一致。

## 10. 检查清单

在提交代码前，请检查：
- [ ] 所有函数都有完整的Doxygen注释
- [ ] 所有宏定义都有适当注释
- [ ] 复杂代码段有解释性注释
- [ ] 临时注释已处理或移除
- [ ] 注释语言统一（中文）
- [ ] 格式符合规范
